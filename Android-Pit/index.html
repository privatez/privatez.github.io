前言：

记录一些在日常开发中碰到的坑

高德地图

  • AMap.clear() 会情况所有地图上的覆盖物

    解决：自定义管理自我管理覆盖物

  • 如何高度自定义 marker？

    解决：采用自定 view + BitmapDescriptorFactory.fromView() 方法实现

    这个方法有个弊端 必须每个 marker 对应一个 view

  • GPS定位出来的地址可能没有地址

    解决：加一层判断 如果没有地址就进行反地理解析经纬度得出地址信息

  • GPS 定位和网络定位经纬度精度不一致

    解决：统一在本地进行一个经纬度的处理成精度为小数后6位

  • 竟可能的在本地做一层封装 这样可以为以后更换其他地图服务省时省力

日常开发

  • 采用自定义 shape 作为 view 的背景有时不起作用

    解决：给 shape 加上一点点 padding

  • LinearLayout设置shape无效。

    这个坑需要满足一下几个条件：

    (1).LinearLayout的orientation属性为vertical

    (2).LinearLayout的background属性为shape

    (3).shape中没有指定padding属性的值

    在以上三种条件满足的情况下shape是不起作用的

  • listview 的 listSelector 属性设置为 seletor 不起作用

    原因：因为 item 布局有一些能获取到焦点的控件

    解决：在 item 布局中的根布局中设置 selector 即可
