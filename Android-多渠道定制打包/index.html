Android 多渠道定制打包

Posted on 2018-07-06 | In Android | Visitors:

记录一些业内比较好的多渠道定制打包方案

美团

基于 apktool 拆包打包

关于 apktool 的介绍可以先看看我这篇文章

  • 首先，在 AndroidManifest.xml 的 节点中添加如下 元素，用来定义渠道的来源：
  • 定义好渠道来源后，接下来就可以在程序启动时读取渠道号

// 在我们的 android 项目根目录下运行命令打包 apk
./gradlew assembleRelease

// 解包后的文件将会保存在 ~/apk-channel/test1 文件夹下
apktool d ‘apk的绝对路径’ -o ~/apk-channel/test1

import re

def replace_channel(channel, manifest):
pattern = r’(<meta-data\s+android:name=”channel”\s+android:value=”)(\S+)(“\s+/>)’
replacement = r”\g<1>{channel}\g<3>“.format(channel=channel)
return re.sub(pattern, replacement, manifest)

// 删除 test1 文件夹下的 original 文件夹（签名文件保存在其中）
// cd 到 ~/apk-channel
// 在解包文件的父文件夹中运行以下命令
apktool b test1

// 1.把你的 keystore 签名文件复制到 apk-channel 文件夹下
// 2.在 apk-channel 下运行一下命令
jarsigner -sigalg MD5withRSA -digestalg SHA1 -keystore your_keystore -storepass your_storepass -signedjar your_signed_apk your_unsigned_apk your_keystore_alias

`
小结：

  • 这种拆包解包的方式可以实现只需要构建一个包。大大减少了构建的时间。但是需要在拆包后修改信息，很容易产生一些失误导致意想不到的错误结果。
  • 通过以上方式推倒出我们可以运用这种方式利用拆包再打包实现替换一些资源文件以实现 app 的定制。具体的实现方式就是在解包后替换掉我们需要修改的资源并重新打包就可。

META-INF

具体实现思路如下：

  • apk 文件重命名为 .zip 文件
  • 解压 .zip 文件后
  • 在 META-INF 文件夹下添加一个名为你需要添加渠道的文件
  • 然后在 Java 文件中去读取这个文件的名字。
